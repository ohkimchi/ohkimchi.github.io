React learning notes

2018-05-12
#Work

  1. Official tutorial
    任何不先读官方资源的框架学习方法都是耍流氓。

  2. React Fundamentals
    The easiest and most updated tutorial you can start with full confidence.

  3. What is React?
    It’s a library for building user interfaces.

  4. What is Element?
    Simply put, a React element describes what you want to see on the screen.
    Not simply put, a React element is an object representation of a DOM node.
    Reasons for why a React Elment isn’t actually the thing you’ll see on your screen, instead, it’s just an object representation of it:
    – JavaScript objects are lightweight. React can create and destroy these elements without too much overhead.
    – React is able to analyze the object, diff it with the previous object representation to see what changed, and then update the actual DOM only where those changes occurred.

  5. What is a component?
    A component is a function or a Class which optionally accepts input and returns a React element.

  6. Reconciliation
    When a component’s props or state change, React decides whether an actual DOM update is necessary by comparing the newly returned element with the previously rendered one. When they are not equal, React will update the DOM. This process is called “reconciliation”.
    Unlike with an HTML tag name, if React sees a Class or a function as the first argument, it will then check to see what element it renders, given the corresponding props. React will continue to do this until there are no more createElment invocations which have a class or a function as their first argument. This whole process is called reconciliation in React and it’s triggered every time setState or ReactDom.render is called.

  7. Notes:
    If you’ve been using React for any amount of time, you don’t use React.createElement to create your object representations of the DOM. Instead, you’re probably using JSX. The primary reason for the confusion is that there’s an often un-talked about abstraction layer between JSX and what’s actually going on in React land. This abstraction layer is that JSX is always going to get transpired to React.createElement invocations (typically) via Babel.

  8. Components, elements and instances
    An element is a plain object describing a component instance or DOM node and its desired properties, and it just describes what you want to appear on the screen in terms of the DOM nodes or other components.
    An instance is what you refer to as this in the component class you write.
    By the end of the reconciliation, React knows the result DOM tree, and a renderer like react-dom and react-native applies the minimal set of changes necessary to update the DOM nodes (or the platform-specific views in case of React Native).

  9. Single-page application
    A SPA is an application that loads a single HTML page and all the necessary assets (such as JavaScript and CSS) required for the application to run. Any interactions with the page or subsequent pages do not require a round trip to the server which means the page is not reloaded.

  10. Bundlers
    Bundlers take JavaScript and CSS code written as separate modules (often hundreds of them), and combine them together into a few files better optimized for the browsers. eg. Webpack & Browserify

  11. Package managers
    Package managers are tools that allow you to manage dependencies in your project. eg. npm and Yarn

  12. JSX
    JSX is a syntax extention to JavaScript. It is similar to template language, but it has full power of JavaScript. JSX gets compiled to React.createElement() calls which return plain JavaScript objects called “React elements”.

  13. props
    props are inputs to a React component, and are readonly. They are data passed down from a parent component to a child component. For components defined as classes, use this.props.children .

  14. state
    A component needs state when some data associated with it changes over time. A component cannot change its props, but it can change its state. To do so, it must call this.setState(). Only components defined as classes can have state.

  15. controlled vs uncontrolled components
    An input form element whose value is controlled by React is called a controlled component.
    An uncontrolled component works like form elements do outside of React.
